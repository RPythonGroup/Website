Git

Git is a local version management system that makes it easy to track and manage changes to the source code being developed. It is not limited to R and is universal.

Github allows you to manage your project version through the concept of cloud-based remote repositories, allowing multiple developers to work on the same project by creating branches to develop independently and then merging the code through Pull Requests.

When working on a statistical project with R, git installation is not absolutely necessary. However, since the example R projects created by the study group are stored in the github remote repository, git installation is recommended.

If you find it difficult to use git, you can think of the github remote repository as a cloud and download the necessary files one by one. For example, you can go to the remote repository address for writing a paper in the study group below.

Repository address for manuscript writing example
https://github.com/RPythonStudy/HANJM

As shown in Figure 1 below, click on the desired file name and then click the Download raw file button in the upper right corner to download it.

However, since the number of files and folders in a project is generally not small, it is difficult to download them individually. Therefore, it is recommended to use it in exceptional situations such as when downloading a small number of files or when conflicts occur when executing git commands.

Git Installation

For installation files and methods by operating system, refer to https://git-scm.com/book/en/v2/Getting-Started-Installing-Git. As of September 27, 2024, the latest installation file for Windows 64-bit is Git-2.46.2-64-bit.exe.

Installation file download site
https://git-scm.com/downloads

To check if the installation is successful, run “git –version” in the command prompt to verify that the version is correct.

Command
git -–version

Next, set the username and email address using the following git command.

uer.name config example
git config --global user.name "BenKorea"
uer.email config example
git config --global user.email "kimbi.kirams@gmail.com"

The local username and email address are not required when downloading example files from the study group’s github remote repository (when using the git pull command). It is only for recording who uploaded the file when uploading your own file (git push command). This is unrelated to the name and email address you used when signing up for github, so you do not need to use the name and email address you used when signing up for github.

uer.name confirm example
git config user.name
BenKorea
git config user.email
[kimbi.kirams@gmail.com]

To start version management, run the git init command in the desired folder (= project folder). Typically, you will be prompted to start using git when you start RStudio. For information on how to use git, please refer to the Korean manual. https://git-scm.com/book/en/v2

In RStudio, there is a setting for whether to use version control interfaces such as git, as shown in Figure 2. You should check this setting so that the option to use git is available when creating a new project.
